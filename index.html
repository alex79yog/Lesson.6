Git comands

git init Команда git init використовується нами для ініціалізації нового Git репозиторію в поточній папці. Вона створює нову підпапку .git, яка містить всі необхідні метадані для репозиторію. Це перший крок для початку відстеження файлів у новому або існуючому проекті з використанням Git.

git status Команда git status дозволяє нам переглянути стан змін у файлах робочого каталогу та області підготовки. Ця команда вказує, які файли не відслідковуються, які змінені, а також які готові до коміту. Вона допомагає нам визначити, які зміни плануємо включити в наступний коміт. Використання git status --untracked-files=all дає нам детальний огляд усіх невідслідкованих файлів, навіть якщо їх багато. Це корисно для повного усвідомлення поточних змін у репозиторії перед їх додаванням до коміту.
git add Команда git add використовується нами для додавання конкретного файла до області підготовки, що означає, що ми плануємо включити зміни цього файла в наступний коміт. Якщо ми використовуємо git add ., ми додаємо всі зміни з поточної папки до області підготовки, включно з новими, зміненими, видаленими файлами, але ігноруючи ті, які перелічені в .gitignore. Це необхідно для визначення які зміни файлів ми бажаємо зафіксувати в наступному коміті.
git commit Команда git commit -m "commit message" використовується нами для створення нового коміту з файлами, які вже знаходяться в області підготовки, з додаванням короткого повідомлення, яке описує зроблені зміни.
git commit -a -m "commit message" робить трохи більше: вона автоматично додає до області підготовки всі відслідковані файли, які були змінені (тобто ті, що вже відомі Git), а потім робить коміт з ними з вказаним повідомленням. Ця команда економить час, оскільки не потребує окремого виконання git add для змінених файлів.
git commit --amend --no-edit використовується нами для зміни останнього коміту без зміни його повідомлення. Це корисно, коли ми хочемо додати забуті файли або виправити помилки в останньому коміті, але не хочемо змінювати вже написане коміт-повідомлення. git commit --amend -m "Новий текст замінить старий текст комміту" також використовується для зміни останнього коміту, але з заміною його коміт-повідомлення на нове. Це дозволяє нам оновити останній коміт, якщо ми помилилися в повідомленні коміту або хочемо його уточнити.
git log Команда git log використовується нами для перегляду історії комітів у нашому репозиторії. git log: Це базова версія команди, яка показує детальну історію комітів, включаючи автора, дату та повідомлення коміту. git log --pretty=format:"%h - %an, %ar - %s": Ця команда налаштовує вивід, показуючи історію комітів у спрощеному форматі. %h - це скорочений хеш коміту, %an - ім'я автора, %ar - відносна дата коміту, %s - повідомлення коміту. git log --since=2.weeks: Виводить коміти, зроблені протягом останніх двох тижнів. Це корисно для перегляду змін за певний період. git log -p -2: Показує останні два коміти разом з відмінностями (patch), внесеними у кожному коміті. Опція p дозволяє нам бачити конкретні зміни (додані чи видалені рядки) в кожному коміті. Ці команди є корисними для відстеження змін у проєкті, аналізу внесків та розуміння історії проєкту.
